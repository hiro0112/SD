### コード変更後の必須手順

**ファイルを追加・変更・削除するたびに、以下の手順を必ず実行すること。**

```powershell
git add <変更ファイル>
git commit -m "変更内容を簡潔に説明するメッセージ"
git push origin main
```

- `git add -A` や `git add .` は `.env` や認証情報など機密ファイルを誤って含めるリスクがあるため、原則として変更ファイルを個別に指定する。
- コミットメッセージは変更の「目的・理由」を一文で表す（例: `"ログイン機能を追加"` / `"バグ修正: null参照エラーを解消"`）。
- プッシュ前に `git status` で意図しないファイルが含まれていないか確認する。

### コミットメッセージのフォーマット

```
<種別>: <変更内容>

例:
feat: ユーザー認証機能を追加
fix: ログイン時のnullエラーを修正
refactor: データ取得処理をサービス層に移動
docs: READMEにセットアップ手順を追記
```

### 注意事項

- `main` ブランチへの `--force` プッシュは禁止。
- 機密情報（APIキー、パスワード、`.env` ファイル）は絶対にコミットしない。
- 破壊的な操作（`git reset --hard`、ブランチ削除など）は実行前にユーザーに確認する。
